<task type="auto">
  <name>Task 1: Product schema and types</name>
  <files>src/lib/db/schema.ts, src/lib/products/types.ts</files>
  <action>
Read first: src/lib/db/schema.ts, src/lib/inventory/types.ts, src/lib/marketplace/types.ts

Add to schema.ts under a "Phase 5: Product Management" section:

1. **productStatusEnum**: pgEnum('product_status', ['draft', 'active', 'inactive', 'deleted'])

2. **products** table:
   - id: uuid PK
   - userId: uuid notNull
   - internalSku: varchar(100) notNull (internal product code)
   - name: text notNull
   - description: text
   - basePrice: numeric(12,2) notNull
   - costPrice: numeric(12,2)
   - categoryId: varchar(100) (internal category)
   - status: productStatusEnum default 'draft'
   - images: jsonb array of { url: string, sortOrder: number }
   - metadata: jsonb Record<string, unknown>
   - createdAt, updatedAt timestamps
   - Indexes: (userId, internalSku) unique, (userId, status)

3. **productVariants** table:
   - id: uuid PK
   - productId: uuid FK -> products.id onDelete cascade
   - sku: varchar(100) notNull (variant-level SKU, links to inventory.sku)
   - optionName: varchar(200) (e.g., "빨강/L")
   - optionValues: jsonb (e.g., { color: "빨강", size: "L" })
   - priceAdjustment: numeric(12,2) default 0 (delta from base price)
   - isActive: boolean default true
   - sortOrder: integer default 0
   - createdAt, updatedAt timestamps
   - Indexes: (productId, sku) unique

4. **productMarketplaceLinks** table:
   - id: uuid PK
   - productId: uuid FK -> products.id onDelete cascade
   - variantId: uuid FK -> productVariants.id (nullable, for variant-level links)
   - marketplaceId: varchar(50) notNull
   - marketplaceProductId: varchar(200) notNull
   - marketplaceCategoryId: varchar(200)
   - syncStatus: varchar(50) default 'synced' (synced/pending/error)
   - lastSyncedAt: timestamp
   - lastSyncError: text
   - rawData: jsonb
   - createdAt, updatedAt timestamps
   - Indexes: (marketplaceId, marketplaceProductId) unique, (productId)

5. **categoryMappings** table:
   - id: uuid PK
   - userId: uuid notNull
   - internalCategory: varchar(200) notNull
   - marketplaceId: varchar(50) notNull
   - marketplaceCategoryId: varchar(200) notNull
   - marketplaceCategoryName: text
   - createdAt, updatedAt timestamps
   - Indexes: (userId, internalCategory, marketplaceId) unique

Create src/lib/products/types.ts with:
- Product interface matching products table
- ProductVariant interface matching productVariants table
- ProductMarketplaceLink interface
- CategoryMapping interface
- ProductFormData (for create/update forms): { name, description, basePrice, costPrice, categoryId, images, variants: { sku, optionName, optionValues, priceAdjustment }[] }
- ProductFilters: { page, pageSize, search, status, categoryId, sort, order }
- ProductSyncStatus type: 'synced' | 'pending' | 'error'
- Korean status labels: PRODUCT_STATUS_LABELS

Per D-01: Products table with internal SKU. Per D-05: Options/variants as separate table with own SKU and price delta.

Read first: src/lib/db/schema.ts (after Task 1), src/lib/products/types.ts, src/lib/inventory/actions.ts, src/lib/inventory/queries.ts

Create src/lib/products/queries.ts:
- getProducts(userId, filters: ProductFilters): paginated list with variant count, uses Drizzle joins
- getProductById(productId): full product with variants and marketplace links
- getProductVariants(productId): all variants for a product
- getProductMarketplaceLinks(productId): all marketplace connections for a product
- searchProducts(userId, query): text search on name/sku for autocomplete

Create src/lib/products/actions.ts with 'use server':
- createProduct(formData: ProductFormData):
  - Insert product row
  - Insert each variant row
  - For each variant, create inventory record via setStock (initial stock 0) linking variant.sku
  - Return created product with variants
  - All in a transaction (per Phase 4 pattern: db.transaction)
- updateProduct(productId, formData):
  - Update product fields
  - Upsert variants (add new, update existing, soft-delete removed)
  - Create inventory for any new variants
  - Per D-05: variant-level inventory tracking
- deleteProduct(productId):
  - Set status to 'deleted' (soft delete)
  - Do NOT delete inventory records (preserve history)
- updateProductStatus(productId, status): change product status

Follow existing patterns from src/lib/inventory/actions.ts for transaction handling and the 'use server' directive.
