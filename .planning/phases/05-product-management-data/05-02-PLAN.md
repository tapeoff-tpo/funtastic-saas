Create src/lib/products/reverse-collect.ts with 'use server':

**reverseCollectProducts(userId: string, connectionId: string, marketplaceId: string)**:
1. Get adapter from registry, authenticate with stored credentials
2. Call adapter.getProducts() to fetch all marketplace products
3. For each NormalizedProduct:
   a. Check if productMarketplaceLinks already has this marketplaceProductId (skip if exists = idempotent)
   b. Create internal product record:
      - internalSku: auto-generate from marketplace prefix + productId (e.g., "CPG-12345")
      - name, description, basePrice from NormalizedProduct
      - status: 'active'
      - images from NormalizedProduct.images
   c. Create variants from NormalizedProduct.variants:
      - sku: auto-generate or use marketplace SKU if available
      - optionName, optionValues, priceAdjustment = variant.price - basePrice
   d. Create productMarketplaceLink connecting internal product to marketplace source
   e. Create inventory records for each variant (initial stock from marketplace if available, else 0)
4. All per-product operations in a transaction
5. Return { imported: number, skipped: number, errors: string[] }

**getCollectionProgress(userId: string)**: Return status for UI polling.

Use the same adapter authentication pattern from order collection worker (read credentials from vault, call adapter.authenticate()).
