1. Create src/lib/shipping/queries.ts with `getHeldShipments(userId: string)`:
   - Join shipments with orders and orderItems
   - WHERE: shipments.shippedAt IS NULL AND shipments.trackingNumber IS NOT NULL AND orders.userId = userId
   - ORDER BY: shipments.createdAt DESC
   - Return: order info (id, marketplaceOrderId, marketplaceId, buyerName, recipientName, status, isHeld, holdReason) + shipment info (trackingNumber, carrierId, carrierName, uploadStatus, createdAt) + items (productName, quantity)
   - Limit to 200 rows (practical cap)

2. Create src/lib/shipping/actions.ts with two server actions:

   a) `reprocessHeldOrder(orderId: string)`:
      - 'use server' directive
      - Auth check via supabase
      - Delete the shipment record for this order (so it goes back to no-tracking-number state)
      - Set order status back to 'preparing'
      - If order isHeld, clear hold (isHeld=false, holdReason=null)
      - revalidatePath('/shipping/held')

   b) `updateHeldMemo(orderId: string, memo: string)`:
      - 'use server' directive
      - Auth check via supabase
      - Update orders.holdReason = memo (reuse holdReason field for memo)
      - Set orders.isHeld = true if not already
      - revalidatePath('/shipping/held')
  </action>
