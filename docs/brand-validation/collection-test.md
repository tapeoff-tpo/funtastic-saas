# FUN-TASTIC Collection Validation Test

Status: designed; no participants or results

## Objective

Identify which continuation mechanisms create a credible additional purchase and distinguish functional demand from aesthetic approval.

Primary hypothesis: H6

Secondary hypotheses: H2, H5

## Collection Conditions

Use the same approved core module, image quality, information density, item count, and displayed price when isolating collection framing.

### C1. Functional Expansion

Adds a new job to an owned Base.

Example statement:

> 현재 Base에 새로운 정리 기능을 추가합니다.

### C2. Zone Expansion

Uses the familiar system in another suitable dry Zone.

Example statement:

> 익숙한 연결 방식으로 다른 생활 공간까지 확장합니다.

### C3. Color Collection

Offers the same approved function in a coordinated color.

Example statement:

> 기존 구성과 조합할 수 있는 컬러입니다.

### C4. Personalization

Adds a name, label, symbol, or selected arrangement.

Example statement:

> 사용하는 사람과 물건에 맞게 표시할 수 있습니다.

### C5. Seasonal Collection

Offers a declared time-bounded palette on an evergreen function.

Example statement:

> 기존 기능은 계속 판매하며, 이번 시즌 컬러를 한정 기간 제공합니다.

### C6. Collaboration Collection

Adds a rights-cleared partner story or palette while preserving compatibility and service.

Example statement:

> 동일한 기능과 호환성을 유지하는 협업 에디션입니다.

No actual partner, licensed artwork, scarcity quantity, or product design is created in this phase.

## Test Design

### Task 1: Fixed-budget choice

Show all six conditions in randomized order after baseline product understanding.

Prompt:

> 동일한 예산으로 하나만 추가할 수 있다면 무엇을 선택하겠나요? 아무것도 선택하지 않아도 됩니다.

Record:

- Choice
- Object or routine affected
- Expected change
- Rejected alternatives

### Task 2: Function trade-off

For participants who choose color, personalization, seasonal, or collaboration:

> 같은 예산으로 새로운 기능 모듈을 선택할 수 있다면 기존 선택을 유지하겠나요?

This reveals whether expression survives an explicit utility trade-off.

### Task 3: Next-purchase sequence

Ask participants to place up to three cards into a likely purchase order and explain the event that triggers each purchase.

Valid triggers include:

- New object
- More capacity
- New Zone
- Better fit
- Gift
- Personal identity
- Seasonal expression
- Service or replacement

“Looks nice” without a trigger remains aesthetic interest, not continuation evidence.

### Task 4: Behavioral proxy

Offer one non-binding action with equal effort for each condition:

- Add to a mock saved list
- Request notification
- Allocate a fixed number of test credits

Label the action as a proxy. Do not describe it as purchase intent or revenue.

## Controls

- Core function remains available in evergreen form.
- No random contents.
- No artificial stock countdown.
- Same visual production quality.
- Same number of pieces where the comparison requires it.
- Same price where testing framing; price is tested separately.
- Collaboration brand familiarity is measured and balanced or excluded.
- Do not use characters or licensed artwork without rights.

## Measures

- First fixed-budget choice
- Nothing-choice rate
- Concrete use trigger
- Choice retained after function trade-off
- Mock saved-list or credit allocation
- Past comparable purchase behavior
- Reason category: utility, cross-Zone, identity, gift, completion, scarcity, social display, other

## Proposed Success Rules

These are proposals, not validated business thresholds.

### Functional or Zone expansion

- Choice is linked to a concrete object, routine, or second Zone
- The participant can state why the existing Base makes the addition more useful
- Interest survives removal of color and edition language

### Color or personalization

- Core function is understood before expression is discussed
- Preference improves among otherwise equally useful choices
- Compatibility and function remain recallable after exposure
- A meaningful subset retains the choice during the function trade-off; required decision threshold is set after pilot

### Seasonal or collaboration

- Interest comes from a relevant story or expression, not confusion about core availability
- Participants understand the function remains available outside the edition
- Service and compatibility expectations remain unchanged

### Failure

- Aesthetic ratings are positive but constrained choices remain “nothing”
- Color is the only remembered product value
- Seasonal or collaboration framing creates false urgency or compatibility confusion
- Participants expect essential functions to disappear
- Follow-up choice exists only when all alternatives are hidden

## Automatic Stop Conditions

- Stop any opaque or random-purchase mechanism for core functional modules.
- Stop a seasonal test if participants believe evergreen function will be unavailable.
- Stop collaboration framing if brand familiarity rather than product relevance dominates the result.
- Stop color expansion if fulfillment or compatibility information becomes less clear.
- Do not approve a collection from likeability ratings alone.

## Pilot and Decision Sample

- Proposed pilot: 8-12 participants per assigned collection framing where monadic exposure is used
- Fixed-budget within-subject task may use the same participant after primary measures
- Decision sample: TBD after pilot choice proportions and minimum meaningful commercial effect are approved

## Results

Not run. No collection mechanism is validated.
