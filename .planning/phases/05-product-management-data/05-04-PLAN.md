Create src/lib/products/excel-import.ts:

**parseProductExcel(buffer: Buffer)**: Parse Excel file into product rows:
- Expected columns: 상품코드(SKU), 상품명, 설명, 판매가, 원가, 카테고리, 옵션명, 옵션값, 옵션가격조정, 옵션SKU
- Rows with same 상품코드 are grouped (base product + variants)
- Return { products: ParsedProduct[], errors: ValidationError[] }
- Validate: required fields (SKU, name, price), numeric values, unique SKUs

**bulkImportProducts(userId: string, products: ParsedProduct[])**:
- For each parsed product:
  a. Check if internalSku exists -> update (PROD-03 pattern)
  b. If new -> createProduct via existing action
  c. If existing -> updateProduct via existing action
- Return { created: number, updated: number, errors: { row: number, error: string }[] }
- Process in batches of 50 to avoid memory issues

Per D-06: ExcelJS for import/export of products with column mapping.

Create src/app/api/products/import/route.ts:
- POST handler accepting multipart/form-data with Excel file
- Auth check (getUser pattern from existing routes)
- Parse file with parseProductExcel
- Execute bulkImportProducts
- Return JSON { created, updated, errors }
