# 품목 양식 및 출고 데이터 반영 설계

## 목표

- 품목 화면의 `ESA009M 업로드` 버튼을 `파일 업로드`로 변경한다.
- ESA009M의 `기존원가(元)` 항목을 화면과 다운로드 파일에서 `특가(元)`로 변경한다.
- 월 판매 계산기 파일의 2026년 4~6월 출고량을 품목별 출고 지표로 반영한다.

## ESA009M 양식

- 내부 표준 헤더는 `특가(元)`만 사용한다.
- 새 파일의 `특가(元)` 열을 직접 읽는다.
- 이전 파일의 `기존원가(元)` 열도 업로드할 수 있도록 읽는 단계에서 `특가(元)`로 변환한다.
- 품목 화면과 품목 다운로드 파일에는 `특가(元)`만 표시한다.
- 품목 업로드는 출고 지표를 변경하거나 삭제하지 않는다.

## 월 판매 계산기 반영

- 원본 파일: `\\tapeoff\homes\tapeoff\9. SCM\3.매출정리\7.구매파일\1. 월 판매 계산기.xlsx`
- 대상 시트: `메인`
- 연결 키: `사방넷상품코드`와 품목의 내부 SKU
- 당월 출고수량: `26/06`
- 3개월 평균 출고수량: (`26/04` + `26/05` + `26/06`) / 3
- 빈 값과 `-`는 0으로 계산한다.
- 품목에 없는 상품코드는 건너뛰고 반영 수와 미매칭 수를 결과로 확인한다.

## 저장 및 조회

- 강제 반영한 출고 지표는 품목 메타데이터에 출처와 기준월을 함께 저장한다.
- 품목 목록과 발주 추천은 저장된 강제 반영값을 우선 사용한다.
- 강제 반영값이 없는 품목은 기존 사방넷 검수 주문 집계값을 사용한다.
- 이후 사방넷 자동 집계 전환 시 출처와 기준월을 이용해 강제 반영값을 안전하게 대체할 수 있게 한다.

## 검증

- 새 `특가(元)` 파일과 이전 `기존원가(元)` 파일이 모두 파싱되는지 테스트한다.
- 저장 결과에서 화면 표준 키가 `특가(元)`인지 테스트한다.
- 4~6월 값의 평균 계산과 빈 값 처리를 테스트한다.
- 품목 목록 및 발주 추천에서 강제 반영값 우선순위를 테스트한다.
- 전체 빌드 후 `main`에 커밋하고 Vercel 운영 배포를 확인한다.
