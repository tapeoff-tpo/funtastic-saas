# FUN-TASTIC Starter Kit Validation Test

Status: designed; no participants or results

## Objective

Determine whether a Starter Kit communicates immediate usefulness and platform expansion more effectively than a single organizer or empty Base.

Primary hypothesis: H3

Secondary hypotheses: H2, H5, H7

## Offer Conditions

All cards use the same typography, dimensions, neutral schematic style, information hierarchy, exposure time, and brand visibility.

### O1. Single Organizer

Label: `책상 소품 트레이`

Included:

- One stand-alone catch-all tray

Message:

> 자주 쓰는 작은 물건을 한곳에 정리합니다.

No Base, module, or expansion claim.

### O2. Base Only

Label: `FUN-TASTIC Base`

Included:

- One Base
- No functional module

Message:

> 필요한 모듈을 선택해 연결하는 시작점입니다. 모듈은 별도 구매입니다.

This condition intentionally tests whether an empty Base has understandable day-one value.

### O3. Desk Starter Kit

Label: `Desk Starter Kit`

Included:

- One Base
- Catch-all tray module
- Pen and tool cup module
- Cable-guide module pack
- Phone-rest module

Message:

> 책상에서 자주 쓰는 물건을 바로 정리하고, 필요할 때 모듈을 추가합니다.

### O4. Universal Starter Kit

Label: `Living Starter Kit`

Included:

- One Base
- Catch-all tray module
- Cup module
- Cable-guide module
- Label module

Message:

> 한 공간에서 시작해, 같은 연결 방식의 모듈을 다른 건식 생활 공간으로 확장합니다.

Do not imply that one identical Base works in wet, hot, portable, or high-load environments.

## Test Sequence

### Round 1: Price-free comprehension

1. Randomly assign one offer.
2. Show the card for three seconds, then remove it.
3. Ask what is included and what can be done immediately.
4. Ask expected first location.
5. Record unaided platform understanding.
6. Ask what, if anything, would be purchased next.

### Round 2: Full-card evaluation

Show the assigned card again without a price.

Measure:

- Immediate usefulness
- Platform comprehension
- Expected use location
- Concrete next module
- Proprietary-standard concern
- Missing information

### Round 3: Relative choice

After primary measures, show all four conditions in randomized order.

- Ask which one the participant would investigate first.
- Ask which one they would reject first and why.
- Ask them to allocate a fixed hypothetical budget across no more than two offers.

The relative choice is a behavioral proxy, not a purchase.

### Round 4: Price research, only after cost inputs

Do not invent prices in this phase.

Before price testing, provide:

- Actual estimated production and fulfillment cost
- Comparable-market price range
- Intended contribution requirement
- Exact package contents

Then test randomized price points or a structured price-sensitivity method. Record displayed prices in the participant data. Do not treat open-ended expected price as willingness to pay.

## Primary Metrics

- Three-second included-content accuracy
- Three-second day-one job accuracy
- Correct platform understanding
- Immediate usefulness rating
- Concrete next-module rate
- First-location clarity
- Proprietary-standard effect
- Offer selected under a constrained choice

## Required Coding

### Platform understanding

- `correct`: understands current included use plus later compatible addition
- `partial`: understands included use but not expansion, or expansion but not current use
- `forced-bundle`: sees extra items without a coherent first job
- `empty-base`: cannot identify day-one value
- `electronic`: assumes connected electronics
- `unsafe-universal`: assumes identical use in every environment

### Next purchase

- `functional-specific`: names module and object or routine
- `capacity`: another Base or expansion capacity
- `identity`: color, label, or personalization
- `vague`: says “more modules” without a job
- `none`

## Proposed Success Rules

These are proposed thresholds, subject to pilot and power review.

### Desk Starter Kit succeeds when

- Correct platform comprehension is at least 70% in the powered test
- It materially outperforms Base Only on comprehension
- Immediate usefulness is not materially worse than Single Organizer
- At least half of correct comprehenders can name a concrete follow-up job without being shown a module list
- Forced-bundle concern does not dominate negative responses

### Universal Starter Kit succeeds when

- Participants name a credible dry second Zone without being led
- Cross-Zone understanding does not increase unsafe universal-Base misconceptions
- Its value is not explained only by receiving more pieces

### Failure

- Starter conditions are understood only after moderator explanation
- Base Only remains empty or speculative in customer language
- Desk Starter is perceived mainly as a forced bundle
- Universal wording increases wet-room or all-environment assumptions
- No concrete next-module job appears after one content revision

## Automatic Stop Conditions

- Stop Base Only as an entry offer if participants cannot state immediate utility.
- Stop Universal Starter messaging if it implies one physical Base works everywhere.
- Stop price comparison if contents, costs, or visual quality differ beyond the intended condition.
- Do not interpret “many pieces” preference as platform understanding.
