    - releaseOrder(orderId) clears hold and restores previous_status

    1. Create `src/lib/orders/queries.ts`:
       - Import db from '@/lib/db', orders/orderItems/claims from '@/lib/db/schema', and Drizzle operators (eq, and, or, ilike, gte, lte, desc, asc, sql, count)
       - Import OrderFilters from './types'
       - Export `getOrders(filters: OrderFilters)`: builds dynamic WHERE clause from filters. Server-side OFFSET/LIMIT pagination (per research: acceptable for admin-only tool). Returns `{ orders: Array<order with items>, total: number }`. Join orderItems via subquery or separate query. Sort by filters.sort (default 'ordered_at') in filters.order direction (default 'desc'). Default pageSize 50.
       - For search filter: use `or(ilike(orders.productName, ...), ilike(orders.buyerName, ...), ilike(orders.marketplaceOrderId, ...))` -- note: productName is on orderItems, so either join or search on buyer/orderNumber only at order level
       - For date filter: `gte(orders.orderedAt, dateFrom)` and `lte(orders.orderedAt, dateTo)`
       - Export `getOrderById(id: string)`: returns single order with items and claims
       - Export `getOrderCount(filters: Omit<OrderFilters, 'page' | 'pageSize'>)`: returns count for pagination

    2. Create `src/lib/orders/actions.ts`:
       - Import db, orders schema, and types
       - Export `updateOrderStatus(orderId: string, newStatus: OrderStatus)`: reads current order, validates transition via isValidTransition, updates status + previousStatus + updatedAt. Returns { success, error? }. Throws/returns error if order is held (per D-11: held orders cannot change status).
       - Export `holdOrder(orderId: string, reason: string)`: validates order exists and is not already held. Sets is_held=true, hold_reason=reason, held_at=now(), previous_status=current status. Returns { success, error? }. (per D-11)
       - Export `releaseOrder(orderId: string)`: validates order is held. Clears is_held, hold_reason, held_at. Restores status from previous_status. Returns { success, error? }. (per D-11)
       - Export `bulkUpdateStatus(orderIds: string[], newStatus: OrderStatus)`: batch status update with per-order validation. Returns { updated: number, errors: Array<{orderId, error}> }.

    Note: For tests that need DB, create mock patterns. The key testable units are: isValidTransition (pure function), filter building logic, and action validation logic. DB-dependent tests can use a test helper that mocks Drizzle.
