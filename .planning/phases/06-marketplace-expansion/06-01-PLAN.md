    Per D-01: 11st uses API key auth with REST/XML endpoints. Per D-05: Use iconv-lite for EUC-KR if needed.

    **client.ts**: Create `createElevenstClient(apiKey: string)` function.
    - Base URL: `https://openapi.11st.co.kr` (11st Open API)
    - Use ky.create() with prefixUrl, same pattern as Coupang client
    - beforeRequest hook: set `openapikey` header to the API key value
    - Set Content-Type to `application/xml;charset=UTF-8`
    - Per D-01: Use fast-xml-parser (already in project deps) to parse XML responses
    - Export a `parseXmlResponse<T>(xmlText: string): T` helper using XMLParser from fast-xml-parser with options: `{ ignoreAttributes: false, attributeNamePrefix: '@_', trimValues: true }`
    - Timeout 30s, retry on 408/429/500/502/503/504 (same as Coupang)

    **types.ts**: Define 11st API response types.
    - `ElevenstOrderResponse`: wraps array of `ElevenstOrder` objects
    - `ElevenstOrder`: ordNo (string), ordPrdSeq (string), prdNm (string), ordQty (number), buyerNm (string), buyerPhone (string), rcvrNm (string), rcvrPhone (string), rcvrZipCd (string), rcvrBaseAddr (string), rcvrDtlAddr (string), ordDt (string), ordStCd (string), selPrice (number), dlvNo (string)
    - `ElevenstClaimResponse`: wraps array of `ElevenstClaim` objects
    - `ElevenstClaim`: clmNo (string), ordNo (string), clmTypCd (string -- CNC/RTN/EXC), clmStCd (string), clmRsnCont (string), clmDt (string)
    - `ElevenstProductResponse` and `ElevenstProduct`: for getProducts
    - `ElevenstInvoiceRequest`: ordNo, ordPrdSeq, dlvMthdCd, dlvCpnyCd, invoiceNo

    **status-map.ts**: Create status mapping functions.
    - `ELEVENST_STATUS_MAP`: Map 11st codes to OrderStatus
      - '202' -> 'new' (결제완료)
      - '301' -> 'confirmed' (상품준비중)
      - '302' -> 'preparing' (배송준비중)
      - '303' -> 'shipped' (배송중)
      - '304' -> 'delivered' (배송완료)
    - `mapElevenstStatus(code: string): OrderStatus` -- fallback to 'new' with console.warn
    - `ELEVENST_CLAIM_TYPE_MAP`: 'CNC' -> 'cancel', 'RTN' -> 'return', 'EXC' -> 'exchange'
    - `ELEVENST_CLAIM_STATUS_MAP`: map 11st claim status codes to 'requested'|'processing'|'completed'|'rejected'
    - `mapElevenstClaimStatus(code: string)` and `mapElevenstClaimType(code: string)` functions

    **adapter.ts**: Create `ElevenstAdapter` class implementing MarketplaceAdapter.
    - Config: id='elevenst', name='11번가', authType='api_key', rateLimitPerSecond=30, requiredCredentials=['api_key']
    - Constructor takes `{ api_key: string }`
    - `testConnection()`: Make lightweight GET request to verify API key is valid
    - `authenticate()`: API key auth has no separate auth flow, return { success: true } (same pattern as Coupang HMAC)
    - `getOrders(since)`: GET /openapi/v3/orders with date range params, parse XML response, normalize each order to NormalizedOrder using status-map
    - `getClaimsOrders(since)`: GET /openapi/v3/claims with date range, parse XML, normalize to NormalizedClaim[]
    - `uploadInvoice(orderId, invoice)`: POST /openapi/v3/orders/{orderId}/delivery with carrier code (via mapCarrierCode('elevenst', invoice.carrierId)) and tracking number. Parse XML response for success/error.
    - `getProducts()`: GET /openapi/v3/products, parse XML, normalize to NormalizedProduct[]
    - `registerProduct(product)`: POST /openapi/v3/products with product data, return marketplaceProductId
    - `updateProduct(id, product)`: PUT /openapi/v3/products/{id}
    - All API methods: wrap in try/catch, throw MarketplaceAuthError on 401/403, throw MarketplaceApiError on other failures (same error pattern as Coupang adapter)
    - normalizeOrder() private method: map ElevenstOrder fields to NormalizedOrder
    - normalizeClaim() private method: map ElevenstClaim fields to NormalizedClaim
