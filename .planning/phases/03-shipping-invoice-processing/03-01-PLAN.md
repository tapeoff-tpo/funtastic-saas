    1. Create `src/lib/shipping/types.ts` with these exports:
       - `InvoiceUploadStatus = 'pending' | 'uploading' | 'uploaded' | 'failed' | 'confirmed'` (per D-02)
       - `InvoiceUploadJobData { orderId, shipmentId, marketplaceId, marketplaceOrderId, connectionId, trackingNumber, carrierId, attempt }`
       - `ShipmentGroup { groupKey, orders, fulfillmentCode, suggestedAction, reason? }` (per D-05/D-06)
       - `CarrierTemplate { id, carrierId, name, columns: Array<{header, field, width, required}>, isDefault, userId }`
       - `CarrierInfo { code, koreanName, englishName }`
       - `ShipmentRecord` matching the DB shipments table shape for query return types

    2. Create `src/lib/shipping/carrier-codes.ts`:
       - `CARRIERS: CarrierInfo[]` with all 14 carriers from research (CJGLS, HANJIN, HYUNDAI, EPOST, KGB, KDEXP, CHUNIL, DAESIN, ILYANG, CVSNET, REGISTPOST, HDEXP, HONAM, ETC)

       - `getCarrierName(code: string): string` returns Korean name
       - `mapCarrierCode(marketplaceId: string, internalCode: string): string` maps internal code to marketplace-specific code. Start with identity mapping (codes are same across Coupang/Naver per research), but structure allows per-marketplace overrides.

    3. Extend `src/lib/db/schema.ts` with Phase 3 tables:
       - `uploadStatusEnum` pgEnum: pending, uploading, uploaded, failed, confirmed
       - `shipmentGroupStatusEnum` pgEnum: suggested, confirmed, rejected, shipped
       - `shipments` table: id, orderId (FK orders), userId, trackingNumber, carrierId, carrierName, uploadStatus (uploadStatusEnum, default 'pending'), marketplaceUploadError, uploadAttempts (default 0), lastUploadAt, shippedAt, createdAt, updatedAt. Index on orderId, index on uploadStatus.
       - `shipmentItems` table: id, shipmentId (FK shipments CASCADE), orderItemId (FK orderItems), quantity (default 1)
       - `shipmentGroups` table: id, userId, groupKey, status (shipmentGroupStatusEnum, default 'suggested'), fulfillmentCode (default 'normal'), maxPackQuantity (integer, default 10), createdAt, updatedAt
       - `shipmentGroupOrders` table: shipmentGroupId (FK shipmentGroups CASCADE), orderId (FK orders), PRIMARY KEY (shipmentGroupId, orderId)
       - `carrierTemplates` table: id, userId, carrierId, name, columns (jsonb), isDefault (boolean default false), createdAt, updatedAt
       - Add `fulfillmentCode` column (varchar 50, default 'normal') to `orderItems` table
