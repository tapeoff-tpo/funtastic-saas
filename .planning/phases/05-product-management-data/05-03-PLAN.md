1. **Add to MarketplaceAdapter interface** in types.ts:
```typescript
registerProduct(product: NormalizedProduct): Promise<{ success: boolean; marketplaceProductId?: string; error?: string }>
updateProduct(marketplaceProductId: string, product: Partial<NormalizedProduct>): Promise<{ success: boolean; error?: string }>
```

2. **Implement in Coupang adapter**:
   - registerProduct(): POST to Coupang product registration API
   - updateProduct(): PUT to Coupang product update API
   - Map internal NormalizedProduct to Coupang-specific product payload
   - Include category mapping (marketplaceCategoryId)

3. **Implement in Naver adapter**:
   - registerProduct(): POST to Naver Commerce product registration API
   - updateProduct(): PUT/PATCH to Naver Commerce product update API
   - Map internal NormalizedProduct to Naver-specific product payload
   - Include category mapping

4. **Create src/lib/products/sync.ts** with 'use server':
   - syncProductToMarketplace(productId, marketplaceId, connectionId):
     a. Load product with variants from DB
     b. Look up category mapping for product's category + target marketplace
     c. Convert to NormalizedProduct format
     d. Check productMarketplaceLinks: if link exists -> updateProduct(), else -> registerProduct()
     e. Update/create productMarketplaceLink with syncStatus and lastSyncedAt
     f. Return { success, marketplaceProductId, error }
   - syncProductToAllMarketplaces(productId):
     a. Load all active marketplace connections for user
     b. Call syncProductToMarketplace for each
     c. Return results array
   - Per D-04: Edit product -> push changes to connected marketplaces

Note: Marketplace API calls may fail due to validation requirements specific to each marketplace. Store errors in productMarketplaceLinks.lastSyncError for debugging.
