  <files>src/lib/jobs/workers/order-collector.ts, worker.ts, tests/jobs/order-collector.test.ts, tests/jobs/claims-collector.test.ts</files>

  <behavior>
    - processOrderCollection fetches credentials from Vault, creates adapter, calls getOrders/getClaimsOrders
    - Orders are UPSERTed with deduplication on (marketplace_id, marketplace_order_id) per D-04
    - Claims are UPSERTed with deduplication on (marketplace_id, marketplace_claim_id) per D-04
    - Order items are inserted alongside orders (cascade)
    - Job execution is logged to job_logs table with orders_collected/claims_collected counts
    - Failed jobs log error_message to job_logs
    - Raw marketplace data is preserved in raw_data column per D-03
  </behavior>
  <action>
    1. Create `src/lib/jobs/workers/order-collector.ts`:
       - Import types: Job from 'bullmq', OrderCollectionJobData from '../queues'
       - Import db, schema (orders, orderItems, claims, jobLogs)
       - Import readCredential from '@/lib/supabase/admin'
       - Import marketplace adapter classes directly (CoupangAdapter, NaverAdapter) — NOT from registry (registry has config-only stubs; worker creates real instances with credentials)
       - Export `async function processOrderCollection(job: Job<OrderCollectionJobData>)`:
         a. Extract { marketplaceId, connectionId, userId } from job.data
         b. Create jobLog entry with status='running', startedAt=now()
         c. Read all required credentials from Vault using readCredential()
         d. Create appropriate adapter instance (CoupangAdapter or NaverAdapter) with credentials
         e. Call `adapter.getOrders(since)` where since = 15 minutes ago (overlap window for safety)
         f. For each normalized order:
            - UPSERT into orders table using `db.insert(orders).values({...}).onConflictDoUpdate({ target: [orders.marketplaceId, orders.marketplaceOrderId], set: { status, marketplaceStatus, rawData, updatedAt } })`
            - Insert order items (delete existing items for order first, then re-insert — ensures item updates are captured)
         g. Call `adapter.getClaimsOrders(since)` (per D-09: claims collected alongside orders)
         h. For each normalized claim:
            - Look up the orderId from orders table using marketplaceOrderId
            - UPSERT into claims table
         i. Update jobLog with status='completed', ordersCollected, claimsCollected, completedAt
         j. On error: update jobLog with status='failed', errorMessage, completedAt
         k. Return { ordersCollected, claimsCollected }
       - Helper: `function createAdapter(marketplaceId: string, credentials: Record<string, string>)`: factory that returns CoupangAdapter or NaverAdapter based on marketplaceId

    2. Create `worker.ts` at project root:
       - Import { Worker } from 'bullmq'
       - Import { connection } from './src/lib/jobs/connection'
       - Import { processOrderCollection } from './src/lib/jobs/workers/order-collector'
       - Import { scheduleAllCollections } from './src/lib/jobs/queues'
       - Create worker: `new Worker('order-collection', processOrderCollection, { connection, concurrency: 2 })`
       - Register event handlers: completed, failed, error
       - Call `scheduleAllCollections()` on startup to ensure all active connections have schedules
       - Add SIGINT/SIGTERM handlers for graceful shutdown (`worker.close()`)
       - Log: "Order collection worker started"
       - Add instructions comment: "Run with: npx tsx worker.ts"
