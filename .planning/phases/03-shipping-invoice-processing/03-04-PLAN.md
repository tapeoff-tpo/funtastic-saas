  <action>
    1. Create `src/lib/shipping/excel/import.ts` per D-03, D-04:
       - `parseInvoiceExcel(buffer: Buffer, columnMapping?: { trackingNumberCol: number, orderIdCol: number, carrierCol?: number }): Promise<ParsedInvoiceRow[]>`
         a. Use ExcelJS to read workbook from buffer
         b. Read first worksheet
         c. Default column mapping: col 1 = order ID, col 2 = tracking number, col 3 = carrier code (configurable per D-03)
         d. Skip header row (row 1)
         e. For each data row, extract { orderIdentifier, trackingNumber, carrierId }
         f. Validate with Zod schema: trackingNumber required (non-empty string), orderIdentifier required
         g. Return { valid: ParsedInvoiceRow[], invalid: Array<{ row: number, errors: string[] }> }

       - `matchInvoicesToOrders(parsedRows: ParsedInvoiceRow[], userId: string): Promise<{ matched: MatchedInvoice[], unmatched: ParsedInvoiceRow[] }>`
         a. Query orders by marketplaceOrderId for the given userId
         b. For each parsed row, find matching order
         c. Return matched (with orderId) and unmatched arrays

    2. Create `src/lib/shipping/excel/export.ts` per D-10:
       - `exportToCarrierExcel(orders: OrderWithShipment[], template: CarrierTemplate): Promise<Buffer>`
         a. Create ExcelJS Workbook
         b. Add worksheet with template.name
         c. Set columns from template.columns (header, key=field, width)
         d. Style header row: bold, gray background (#E0E0E0), border
         e. Add data rows by extracting values using `getNestedValue(order, field)` helper
         f. Handle nested fields like `shippingAddress.address1` with dot notation
         g. Return `Buffer.from(await workbook.xlsx.writeBuffer())`

       - Helper: `getNestedValue(obj: Record<string, unknown>, path: string): unknown` -- resolve dot-notation paths

    3. Create `src/lib/shipping/excel/order-export.ts` per DATA-01:
       - `exportOrdersToExcel(orders: OrderRow[], columns: Array<{ field: string, label: string }>): Promise<Buffer>`
         a. Similar to carrier export but uses AVAILABLE_ORDER_FIELDS for column options
         b. User selects which columns to include (passed as `columns` param)
         c. Styled headers, auto-width
         d. Return buffer

    All Excel functions are server-side only (no client-side parsing per research anti-pattern).
