# Project FUN-TASTIC - Universal Living Platform Report

Checked: 2026-07-18

## Executive Summary

FUN-TASTIC can pursue a Universal Living Platform, but the viable universal element is **the interface and brand experience**, not one identical physical Base or one material in every room.

The evidence from IKEA SKADIS, KUNGSFORS, BOAXEL, String, Gridfinity, Multiboard, openGrid, Bubbluu BLOCK and premium desk ecosystems supports four conclusions:

1. Customers understand systems that combine a Base with separately useful accessories.
2. Hooks, trays, containers, shelves and holders recur across Zones.
3. Existing brands usually change the Base or mounting system when room, load, moisture, or furniture context changes.
4. Open FDM ecosystems can create a large accessory long tail, but fit, terminology, discoverability, and compatibility governance become major risks.

The strategic architecture should therefore be:

`one interface protocol + multiple validated Base classes + universal module families + Zone Starter Kits + one compatibility promise`

**Final verdict: GO WITH LIMITATIONS.** Launch and validate a dry-zone platform first. Treat all-Zone coverage and 1000 SKUs as staged options, not current product claims.

## 2. Zone Map

Thirteen Zones define the intended living network:

| Zone | Core job | Environment | PLA-first status |
|---|---|---|---|
| Study | Learn, write, read and manage stationery | Dry, low load | Core |
| Work | Dock devices and daily work tools | Dry, low load | Core |
| Bedroom | Create a bedside and dressing landing place | Dry, low load | Core |
| Kids | Make storage visible, understandable and personal | Dry, low load; safety review | Core expansion |
| Living Room | Group remotes, controllers, charging and display items | Dry, low load | Core expansion |
| Beauty | Organize dry cosmetics, brushes and jewelry | Dry if outside bathroom | Core expansion |
| Coffee | Stage dry preparation tools | Local heat and splash risk | Conditional |
| Entryway | Manage arrival and departure objects | Mounting and variable load | Conditional |
| Pet | Group light walking, care and training items | Dirt and variable load | Conditional |
| Travel | Carry a compact routine | Drop, mobility and loss risk | Different Base class |
| Kitchen | Organize cooking and cleaning tools | Heat, water and food contact | Dry sub-zone only in PLA |
| Laundry | Manage sorting, repair and care tools | Humidity and chemicals | Conditional/future material |
| Bathroom | Organize wet grooming tools | Sustained humidity and water | Not PLA MVP |

`zone-map.csv` lists 50 small objects for each Zone, 650 entries total.

## 3. Repeating Object Jobs

The 650 objects do not imply 650 fundamentally different products. Their storage jobs repeatedly collapse into a smaller module vocabulary:

- Contain: tray, cup, bin, pocket
- Separate: divider, indexed rack, slot, ring
- Suspend: hook, loop, hanger
- Retain: clip, catch, cradle, collar
- Present: stand, riser, ledge, card holder
- Route: cable guide, roll holder
- Identify: label and nameplate
- Move: portable carrier
- Maintain: blank cover, adapter, replacement detent

`universal-module-map.csv` defines 35 reusable module families and their Zone reach. This is a taxonomy, not proof that one geometry fits every object.

## 4. What Can Be Commonized?

### Common across almost all Zones

- Interface protocol
- Attachment and release interaction
- Compatibility mark and versioning
- Label system
- Tray, cup, hook, clip, divider, holder and pocket vocabulary
- CMF and packaging language
- Catalog metadata
- Service and replacement logic

### Must vary by Zone or environment

- Base mounting method
- Base material
- Drainage and cleaning features
- Heat, chemical, impact and moisture claims
- Load class
- Object-fit geometry
- Portable versus stationary form

The correct commonization target is therefore a **platform grammar**, not an identical object in every room.

## 5. Common Base Coverage

A single dry stationary PLA Base can credibly serve six Zones without changing its environment promise:

- Study
- Work
- Bedroom
- Kids
- Living Room
- Dry Beauty

It can conditionally enter Coffee-dry, Entryway, Pet-light, Kitchen-dry and Laundry-dry after local risk validation. Travel requires portability; Bathroom requires wet-zone material and cleaning validation.

Answer to the core question:

- One identical physical Base: **6 direct Zones, 5 conditional Zones, 2 requiring another Base class**
- One interface protocol: **all 13 Zones are strategically addressable**

## 6. Zone Starter Kit Strategy

Each Starter Kit should solve one complete job and expose a specific next purchase. The full assortment is in `starter-kit-strategy.md`.

Recommended first sequence:

1. Study Starter
2. Work Starter
3. Bedroom or Beauty bridge
4. Living Room or Kids bridge
5. Conditional Coffee, Entryway, and Pet pilots
6. Portable Travel class
7. Wet and humid Zones after material expansion

Do not launch thirteen kits simultaneously. That would multiply unverified Base, packaging, instruction, color, and support combinations before repeat behavior is known.

## 8. Why Customers Add Modules

Evidence from existing platform research supports these repeat-purchase motives:

- New objects need a place.
- The first configuration reveals another clutter point.
- Base capacity expands.
- Generic storage is replaced by a better-fit module.
- A familiar system is extended to another room.
- Color, display, and identity become part of the setup.
- Replacement or unavailable parts are needed.

FUN-TASTIC-specific repurchase behavior remains unverified. The first commercial proof should be a follow-up module order, not social-media interest alone.

## 9. Brand Philosophy

**One familiar connection, wherever life happens.**

FUN-TASTIC should differ from conventional organizing brands through continuity, pleasurable attachment, curated expansion, on-demand variety, and long-term compatibility stewardship.

The brand world is not “a house full of matching bins.” It is a personal living map in which familiar modules move and adapt as routines change.

Customers should continue buying because prior modules remain useful and new modules solve real adjacent jobs. Proprietary lock-in without module breadth, availability, and backward compatibility would destroy that value.

The full principles are in `brand-philosophy.md`.

## 10. Platform Architecture

Seven layers separate what must remain universal from what must adapt:

1. Interface protocol
2. Environment-specific Base class
3. Shared module chassis
4. Functional module
5. CMF system
6. Catalog and compatibility profile
7. Zone Starter Kit

Phase 5's guided slide plus terminal detent remains the primary interface research direction, with hook-and-slide and short-turn bayonet retained as physical comparators. No proprietary geometry is frozen.

## 11. 1000-SKU Expansion

The platform has a logical 1000-SKU capacity:

| Portfolio layer | Capacity |
|---|---:|
| Base and mounting classes | 20 |
| Universal module families | 80 |
| Zone-core modules | 390 |
| Object-fit and size variants | 200 |
| Validated color/material variants | 150 |
| Personalization and collaborations | 100 |
| Adapters, replacements and service | 60 |
| **Total** | **1000** |

This does not justify stocking 1000 finished goods. The viable model is a small stocked core plus on-demand manufacturing, controlled color/material variants, catalog metadata, and retirement governance.

## 12. Expansion Risks

1. Universal-interface ambition can force a mechanically compromised connector.
2. One physical Base cannot honestly serve dry, wet, hot, portable, mounted, and high-load contexts.
3. Thirteen simultaneous Starter Kits would create a severe cold-start and support burden.
4. Small follow-up orders may be uneconomic to ship.
5. A proprietary system can lose trust if modules disappear or compatibility changes.
6. Color and fit variants can create 1000 SKUs without creating 1000 useful products.
7. PLA assumptions can leak into inappropriate wet or high-temperature uses.
8. A broad catalog can become impossible to understand.
9. FDM competitors can copy simple modules.
10. Social sharing can overstate actual repeat purchase.

## 13. Evidence Required Before Expansion

- Paid Starter Kit conversion
- First and second module attach rate
- Time to follow-up purchase
- Cross-Zone reuse or second-Base purchase
- Small-order shipping contribution margin
- Compatibility-related support and return rate
- Active modules per Base
- Module relocation behavior
- Willingness to trust a proprietary compatibility promise
- Wet-zone demand sufficient to justify a new material and Base class

## 14. Final Verdict

# GO WITH LIMITATIONS

### GO

- Define the brand and catalog around one interface protocol.
- Use Study and Work as the first proof, then test one cross-Zone bridge.
- Build a universal module vocabulary and Zone-specific Starter Kits.
- Maintain a path to dry mounted, portable, and wet Base classes.

### LIMITATIONS

- Do not claim one physical Base works everywhere.
- Do not launch wet, hot, high-load, or safety-critical modules in the PLA-first MVP.
- Do not freeze the interface before physical Phase 5 comparison.
- Do not pursue 1000 active stocked SKUs without demonstrated repeat and cross-Zone behavior.
- Do not confuse color variants, social engagement, or digital catalog size with platform adoption.
